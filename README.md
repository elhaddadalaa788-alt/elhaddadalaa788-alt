## 🔐 Featured Projects

### Debian Server Hardening & Incident Response Project
This project focused on investigating and hardening a vulnerable Debian Linux server in a lab environment.

Key work included:

- Conducting network scanning with **Nmap** to identify exposed services including FTP, SSH, SMTP, HTTP, IPP, and MySQL
- Reviewing attack surface exposure and identifying unnecessary services
- Applying **UFW** firewall rules to restrict non-essential ports and reduce external exposure
- Hardening **SSH** by disabling direct root login
- Securing sensitive files such as `wp-config.php` using `chmod 600`
- Running **rkhunter** to support rootkit and integrity checks
- Reviewing Linux system logs to support investigation and technical documentation

This project helped strengthen my understanding of how security teams approach system hardening, containment, and defensive remediation in a structured way.

---

### OSINT & External Exposure Analysis Lab
This project focused on public exposure research and open-source intelligence methods.

Key work included:

- Using **Google Dorks**, **Shodan**, and **Have I Been Pwned**
- Reviewing exposed services and public-facing security risks
- Checking for credential exposure and understanding attack surface visibility
- Analyzing how public information can reveal security weaknesses
- Documenting findings in a clean and structured format

This work helped me better understand how attackers and defenders both use publicly available information, and why OSINT matters in early-stage security analysis.
